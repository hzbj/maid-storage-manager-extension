# 03 女仆接送策划与实施规则

> 状态：终端页面、司机调度、自动驾驶、控制交接和恢复状态机已实现
> 共用规则见 [女仆物流玩法总纲](../GAMEPLAY_DESIGN_MAID_LOGISTICS.md)。

## 1. 功能目标

女仆接送直接服务玩家：玩家在女仆联络终端的实际地形地图上选择起点和终点，呼叫账号名册中的空闲快递员女仆骑扫帚接客。接送不生成快递箱，不新增载客道具，也不转移女仆背包里的扫帚。

## 2. 账号与司机

- 终端账号保存共享女仆名册，并分别保存默认网络仓库快递员和默认司机。
- 同一女仆可以兼任两个默认角色，但货运、杂物整理和接送事务严格互斥。
- 接送左栏列出账号中已注册的司机候选，显示加载状态、当前职业、是否有扫帚及真实工作阶段。
- 开始订单时服务端重新确认司机属于账号、当前使用快递员职业、位于主世界、持有扫帚且没有其他事务。

## 3. 地图选点

- 未手动设置起点时，使用玩家点击“开始接送”时的位置；该位置必须可见天空。
- 手动选点只提交 X/Z，服务端负责寻找地表高度以及 12 格内平整、露天、净空的安全位置。
- 终点允许位于客户端尚未探索、没有地形纹理的区域；坐标仍可在未知底图上选择，服务端在航线执行时加载目标区块并解析地表。
- 短按地图设置当前选择的起点或终点；拖动累计超过 4 像素后只平移地图，不提交选点。
- 起点和终点使用明确的“起/终”标记，虚线只在接送页出现，并带“接送路线”图例。

## 4. 接送状态机

1. **前往起点**：司机复用现有女仆寻路、安全起飞、区块票据和扫帚避障规则前往起点。
2. **等待主人**：司机作为第一乘客骑在临时服务扫帚上；只有本次下单玩家能成为第二乘客。
3. **自动飞行**：玩家上车后自动起飞，服务端控制扫帚，客户端玩家输入不能操纵。
4. **正常抵达**：在终点 12 格内找到安全落点后自动降落，玩家与女仆下车，服务扫帚回收。
5. **玩家接管**：自动飞行中点击“结束行程”，或终点附近没有安全落点时，乘坐顺序切换为玩家第一、女仆第二，解除自动驾驶并恢复上游扫帚原生控制。
6. **下车恢复**：玩家接管后下车、死亡或断线，女仆重新成为第一乘客，接管服务扫帚并寻找附近安全地面，落地后回收。

尚未上车时点击“结束行程”按取消订单处理。紧急降落阶段不允许再次交接控制。

## 5. 高空航线与未知终点

- 货运扫帚原有巡航规则保持不变；载客航线单独使用 **56 格基础巡航净空**。
- 飞行过程中持续读取前方地形高度，至少保持 **24 格动态地形净空**；检测到扫帚、女仆或玩家碰撞箱受阻时继续抬升或悬停，不穿墙推进。
- 未知终点不要求客户端预先加载或探索。服务端区块票据逐步加载目标与落点搜索区，加载完成前保持当前飞行状态。
- 落点搜索半径固定为 **12 格**。搜索耗尽不把玩家困在空中等待，也不瞬移落地，而是自动把扫帚交给主人操纵。

## 6. 临时服务扫帚

- 服务扫帚是运行时实体，不对应新的物品，也不消耗、复制或移动司机库存中的扫帚。
- 自动驾驶阶段带有司机与指定乘客标记；非下单玩家右键会被服务端拒绝。司机在等待时会遮住部分扫帚碰撞箱，因此右键司机或右键服务扫帚都会进入同一条服务端授权登乘流程。服务端专用持久标记不依赖客户端同步，两端都以已同步的“女仆为第一乘客”顺序屏蔽原版扫帚移动和玩家输入。
- 玩家接管时移除自动驾驶标记，把乘坐顺序调整为“玩家第一、女仆第二”，再恢复原生物理与控制；下车后重新加回自动驾驶标记完成安全降落。
- 正常抵达、取消或恢复落地后销毁服务实体，女仆库存保持不变。

## 7. 失败与限制

| 情况 | 处理 |
| --- | --- |
| 玩家、司机不在主世界 | 拒绝开始；在途异常则女仆接管并降落 |
| 默认起点上方遮挡 | 拒绝开始并显示原因 |
| 司机离线、换职业、缺扫帚或忙碌 | 拒绝开始，不占用其他司机 |
| 自定义点尚未加载 | 允许提交；由服务端加载并解析 |
| 起点无安全位置 | 取消尚未开始的乘坐，不生成扫帚物品 |
| 终点无安全位置 | 自动交给下单玩家控制 |
| 非下单玩家登车 | 服务端拒绝 |
| 玩家中途下车、死亡或断线 | 女仆接管并寻找最近安全落点 |
| 扫帚实体异常消失 | 清理接送事务，不修改女仆库存 |

## 8. 首版边界

- 只支持主世界、单名下单玩家和单名司机。
- 不做费用、评价、拼车、多乘客或跨维度载客。
- 不要求信箱作为起终点；信箱仍服务网络仓库和快递货运。
- 接送事务不锁住账号中的其他快递员，也不计入仓库库存事务。

## 9. 验收标准

- 顶部只显示“网络仓库”和“女仆接送”。
- 司机左栏能区分空闲、缺扫帚、离线、其他工作和接送各阶段。
- 已探索与未知区域都可选为终点，地图缩放、平移和起终点标记始终与地形对齐。
- 自动飞行时玩家无法控制；主动结束或无落点交接后可以控制。
- 正常抵达自动落地；下车、死亡、断线能够进入安全恢复。
- 全过程不消耗、复制或转移女仆库存中的扫帚。
