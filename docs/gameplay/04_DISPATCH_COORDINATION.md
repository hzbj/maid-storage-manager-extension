# 04 派遣与职业编排策划案

> 状态：仅拆分，等待前三项的公共物流基础稳定后再确认
> 共用规则见 [女仆物流玩法总纲](../GAMEPLAY_DESIGN_MAID_LOGISTICS.md)。

## 1. 功能目标

派送与职业编排服务于女仆之间的长期协作：玩家在女仆联络终端中设置职业、工作范围、工作时间和货物目标，职业女仆把产物打包成快递箱，快递员将其送给下一名女仆继续处理。

## 2. 地图配置

玩家在“派送”模式中可以：

- 选择一名已登记女仆；
- 查看其来源模组和当前职业；
- 选择适配器确认支持的工作类型；
- 框选工作范围并填写 Y 范围；
- 设置始终、白天、夜晚或自定义游戏内时间段；
- 选择工作结束后的行为；
- 绑定该职业女仆的快递信箱；
- 设置产物的目标信箱/目标女仆；
- 为一条或多条连接分配共享快递员。

## 3. 职业女仆打包

- 职业适配器声明哪些物品属于已完成产物。
- 达到触发条件后，女仆前往自己的快递信箱存放区。
- 系统按目标和 14 格容量拆分，生成一个或多个快递箱。
- 每个箱只指向一个目标女仆、信箱或玩家。
- 存放区已满时女仆保留物品并等待，不丢弃、不强行塞入快递员背包。

可选触发条件：

- 一轮工作完成；
- 女仆背包达到设定比例；
- 指定物品达到数量；
- 固定游戏时间；
- 玩家手动要求回收；
- 目标女仆发出补给需求。

## 4. 连接与共享快递员

一条连接保存：

- 来源女仆/信箱；
- 目标女仆/信箱；
- 允许打包的物品；
- 触发条件；
- 优先级；
- 可服务的快递员池。

一个快递员可以负责多个连接。各连接产生快递箱后进入队列，快递员空闲时按优先级、等待时长、距离和是否能顺路满载选择下一单。

同一快递员同一时间只运行一个会修改其背包、乘骑或在途账本的实体事务。首版不做复杂多箱拼车；确认单箱闭环稳定后再扩展批量运输。

## 5. 目标女仆接收

- 仓管：接收快递箱并把货物整理到真实仓库。
- 厨师：接收食材箱，把允许原料交给当前烹饪工作。
- 砍树/挖矿：可以接收工具、燃料、食物或附属声明的补给。
- 其他职业：只能处理适配器声明的输入，未知物品保留在快递箱。

目标女仆完成下一步后，可以继续生成指向第三名女仆的快递箱，从而形成生产链。

## 6. 伐木到仓库示例

1. 玩家选择砍树女仆并框选林场。
2. 设置白天砍树、背包达到 70% 时打包。
3. 为砍树女仆配置快递信箱、起落区和存放区。
4. 建立“林场信箱 → 仓库信箱”的连接。
5. 把一名快递员加入该连接；同一快递员还可负责矿区与农场连接。
6. 砍树女仆把原木打成目标为仓管的快递箱。
7. 快递员运输到仓库信箱。
8. 仓管接收并按现有规则分类入库。

## 7. 兼容策略

- 深度适配：终端可安全设置职业、范围、时间、打包和接收。
- 协调适配：终端只配置物流；工作区和职业仍在原附属界面设置。
- 只读兼容：只显示女仆、职业和位置，不自动取物或投递。

任务使用 `sourceModId + taskResourceId` 标识。附属移除或版本不兼容时暂停连接并保留原数据，不把女仆自动切换到同名任务。

快递员不能默认抽取职业女仆整个背包；只有适配器确认的产物或玩家指定输出库存可以打包。

## 8. 首版边界

- 首个外部职业只选一个明确版本的砍树附属做纵切片。
- 首版只支持矩形工作区和游戏内每日时间段。
- 首版每个信箱只绑定一名职业女仆。
- 首版不做循环生产配方、成本优化或跨维度生产链。
- 不新增工作区旗、职业印章、餐盒或其他专用道具。

## 9. 待后续确认

2. 工作范围由终端覆盖附属设置，还是只在附属支持时同步。
3. 快递员是否允许一次携带多个同路线快递箱。
4. 目标女仆长时间不接收时，默认等待还是退回来源。
5. 排班边界到达时，是完成当前动作再停工，还是完成整轮工作再停工。
