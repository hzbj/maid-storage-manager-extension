# 女仆物流 玩法策划案

## 一、定位

与"网络仓库""女仆接送"同级，终端新增第三入口 **「女仆物流」**，用于规划快递员在多个仓库、营业点之间的自动化循环运输。

---

## 二、新道具：营业许可证

### 2.1 核心属性

| 属性 | 说明 |
|------|------|
| 悬挂方式 | 必须挂在墙上 |
| 绑定女仆 | 最多 6 名，必须同一职业（均为快递员） |
| 物品规则 | 白名单或黑名单（二选一） |
| 生效范围 | 许可证所在位置水平 64 格内的箱子 |
| 起落区 | 潜行右键地面设置快递员起落区 |

### 2.2 操作

| 操作 | 功能 |
|------|------|
| 手持右键 | 打开物品配置界面，设置白名单/黑名单，可设准放物品列表 |
| 潜行右键箱子 | 将该箱子注册为许可证管辖容器（可注册多个） |
| 潜行右键地面 | 设置快递员的起飞/降落区（露天、3×3 平整） |
| 右键女仆 | 打开绑定界面，选择要绑定的快递员女仆 |

### 2.3 约束

- 许可证管辖范围内的箱子才受物品规则约束。
- 快递员向许可证管辖箱投放物品时，只允许放入白名单物品 / 拒绝黑名单物品。
- 不限制从管辖箱取货——只管控"放"不管控"拿"（取货由请求列表等既有机制处理）。

## 三、女仆物流界面

### 3.1 布局


- 左栏菜单取消，全部内容并入地图区域。
- 地图上显示：快递员女仆位置（带名字）、已注册仓库、所有营业许可证。
- 点击地图上任意节点，右栏显示其详细信息。

### 3.2 右栏点击展示内容

**点击女仆快递员：**
- 当前状态（空闲 / 运输中 / 等待起降）
- 正在运输的物品名称与数量
- 当前路线起点 → 终点
- 所属营业许可证

**点击仓库：**
- 仓库名称
- 当前库存物品种类数
- 是否满仓（物品无法放入时标黄）

**点击营业许可证：**
- 物品规则：白名单 / 黑名单 + 具体清单
- 绑定的快递员列表（在岗 / 离岗）
- 管辖容器数量
- 线路列表（起点→终点，调度次数）

## 四、线路系统

### 4.1 创建线路

1. **选起点**：点击地图上的营业许可证 或 仓库
2. **选终点**：点击另一个节点（与起点不同）
3. 两点之间出现**带箭头的虚线**，箭头指示运输方向（颜色与分配的快递员绑定）
4. **确认**：右栏出现操作面板，可分配快递员女仆、设置调度 → 点击「确认连接」生效
5. **取消**：点击地图空白区域取消选择

> 若需要 A 运到 B、B 也运到 A，再建一条 B→A 线路即可，两条独立线路各自计次调度，不是单独的功能开关。

### 4.2 线路成立条件

线路必须有发有收才能成立，否则「确认连接」灰色不可点击：

- **起点是营业许可证** → 出货物资受该证管理（可放入管辖容器的物品才能发出）。终点必须有有效仓库或另一个许可证的接收箱。
- **起点是仓库** → 物资从仓库出发。终点必须有许可证管辖箱接收。
- **终点是营业许可证管辖箱** → 只有该证白名单内的物品 / 非黑名单物品才能接收。

简言之：**发得出、收得了，线路才成立**。

## 五、调度系统

### 5.1 次数

- 每条线路每天设一次数（如 2 次/游戏日）。
- 多线路按玩家设定的顺序执行，直到当天全部线路的次数用完。
- 默认顺序为添加顺序，玩家可在右栏拖拽调整，如 A(2) B(2) C(2) → AABBCC 或 ABCABC 或 ACBABC 等。

### 5.2 跳过一次

以下情况**跳过本次运输，不消耗次数**：
- 起点箱子（或管辖范围内）没有任何目标物品可取。
- 终点箱子已满，无法放入任何一件目标物品。

### 5.3 满仓（红线）

- 单次运输因接收地已满无法放入物品时，该线路变为**红色带箭头虚线**。
- 首次触发时终端右侧显示红色警告：「[起点→终点] 接收地已满，暂时跳过」；之后同线路继续满仓不重复通知。
- 发出地无货**不走红线**——那是灰色（5.6）。红线仅表示"接收端拒收"。

### 5.4 货物数量触发

- 营业许可证可设置「物品触发规则」：当管辖范围内某物品的 **总数量 ≥ N** 时，为该线路的快递员额外 **+1 次**当日运输。
- 每个物品独立触发，额外次数在当前日有效。

### 5.5 每日结算

- 游戏日结束时（0:00 或黎明），当天线路的计时器清零。
- 未完成的次数**不累计**到第二天。
- 当天线路全部完成 → 地图上该线路变**灰色**。
- 若当天有次数未完成（无论原因）→ 该线路变**黄色带箭头虚线**，表示分配的快递员无法胜任该线路的全部任务，终端提醒：「[线路] 今日未完成运输，建议减少次数」。

### 5.6 空闲跳过

- 若管辖容器范围内没有任何目标物品，该线路所有次数在图上一开始即标灰，快递员保持当前任务等待下一轮，不消耗次数不移动。
- 不会出现「空手跑一趟又空手回来」的情况。

## 六、地图视觉规则

| 元素 | 标记 | 说明 |
|------|------|------|
| 快递员女仆 | 圆点（颜色与所属许可证绑定） | 同一许可证下所有快递员同色 |
| 仓库 | 🏠 图标 | 悬停显示名称、坐标、库存数 |
| 营业许可证 | 📋 图标 | 悬停显示名称、绑定人数 |
| 运行中的线路 | 同色带箭头虚线 | 快递员所属许可证的颜色，箭头指向运输方向 |
| 满仓（单次） | 红色带箭头虚线 | 接收地已满，该次运输无法放入 |
| 超载（全天） | 黄色带箭头虚线 | 当日次数未全部完成，快递员无法胜任该线路任务 |
| 已结束 | 灰色带箭头虚线 | 次数用尽或发出地无货 |

- 选中（创建线路中）的箭头虚线为高亮闪烁，两端点各有一个强调圆点。

## 七、边界与约束速查

1. 营业许可证必须挂在墙上，管辖箱必须在 64 格内。
2. 每个许可证最多 6 名快递员，所有快递员必须是同一职业。
3. 线路必须有有效收发两端（不能只发不收或只收不发）。
4. 快递员忙（正在执行其他快递/接送任务）时，线路任务排队等待。
5. 当天全部线路已完成 → 快递员回到正常空闲/跟随状态。
6. 源箱无货跳过不耗次数，目标箱满跳过不耗次数，不强制失败重试。
7. 满仓仅对首次发生提醒一次。
8. 第二天不补昨日未完成次数。
9. 物品触发额外次数每天独立计算。
10. 一个箱子可以同时属于多个许可证的管辖范围（物理上），但线路创建时会按许可证规则校验收发合规。
